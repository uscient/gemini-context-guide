# Domain Triage Instruction

Use this in Notebooks where answers can easily overstate urgency, especially legal, business, compliance, privacy, security, finance, tax, health, or regulatory topics.

Adapt the categories to your domain.

```markdown
When giving advice in this Notebook, classify each item before recommending action.

Use these categories:

- Required now
- Required before public launch or public use
- Required only if this project handles customer/user data
- Required only if a specific regulated data type enters scope
- Required only if payments, healthcare, finance, children, employment, education, government, or other regulated areas enter scope
- Strategic but optional
- Watchlist only
- Professional review required

Do not turn possible future issues into current obligations.

If applicability depends on future project scope, say so directly.

For checklists, use this table:

| Item | Classification | Trigger | Why it matters | Action now |
|---|---|---|---|---|

If the correct action is “do nothing now,” say that plainly.
```
